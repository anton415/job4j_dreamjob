# Задача:
Добавить возможность загрузки файла на сервер.

## Что нужно:
1. Создать файл src/main/resources/application.properties:
    - Указать директорию, где будут храниться файлы: files;
    - Указать максимальный размер файла: 10MB.
2. Создать доменную модель File и реализовать для нее репозиторий и сервис;
3. Создать DTO (Data Transfer Object) класс FileDto;
4. Создать контроллер FileController;
5. Связать File и Vacancy;
6. Обновить MemoryVacancyRepository;
7. Обновить VacancyService и SimpleVacancyService;
8. Обновить view: create.html, one.html;
9. Обновить VacancyController;
10. По аналогии добавить возможность загрузки файлов для резюме кандидатов при создании, редактировании. Для хранения также использовать FileService. При удалении резюме связанные файлы также нужно удалять;

## Контекст:
- Доменную модель File(String name, String path);
- Репозиторий и сервис должны содержать следующие методы:
    - File save(File file);
    - Optional<File> findById(int id);
    - void deleteById(int id);
- ДТО содержит структуру FileDto(String name, byte[] content);
- @Value("${file.directory}") String storageDirectory. Эта строка позволяет подставить на место storageDirectory значение из файла application.properties с ключом file.directory;
- storageDirectory + java.io.File.separator + UUID.randomUUID() + sourceName. Так создается уникальный путь для ногового файла. UUID это просто рандомная строка определенного формата;
- В контроллере FileController если файл не найден по id, то клиенту возвращается статус 404, а если найден, то статус 200 с телом ответа в виде содержимого файла. 
- Не помещать все из File в Vacancy, потому что это будет нарушать SRP для Vacancy.
- При удалении вакансии удаляется связанный с ней файл;
- При сохранении и обновлении вакансии сохраняется связанный с ней файл. 
- <input type="hidden" name="fileId" th:field="*{fileId}"> нужна, чтобы при обновлении с пустым изображением мы не сохраняли старое изображение;
- <img th:src="@{/files/{fileId}(fileId=*{fileId})}" class="w-100" alt="No image"> служит для вывода изображения.
- @RequestParam MultipartFile file - так мы получаем файл из формы. Название параметра соответствует name из формы;
- new FileDto(file.getOriginalFilename(), file.getBytes()) - так мы передаем "упакованные" в DTO данные для обработки в сервисе.
