# Задача:
Подключение к базе.

## Что нужно:
1. Добавить зависимости:
    - spring-boot-starter-thymeleaf
    - spring-boot-starter-web
    - spring-boot-devtools
    - postgresql
    - commons-dbcp2
    - sql2o
2. Создать БД dreamjob
3. Создать папку db/scripts
4. Создать скрипт db/scripts/001_ddl_create_cities_table.sql в котором будет таблица cities
5. Создать скрипт db/scripts/002_dml_insert_cities.sql в котором будет добавление трех городов: Москва, Санкт-Петербург, Екатеринбург
6. Создать скрипт db/scripts/003_ddl_create_files_table.sql в котором будет таблица files
7. Создать скрипт db/scripts/004_ddl_create_vacancies_table.sql с таблицей vacancies
8. Выполнить скрипты
9. Внести строки для подключения к БД в application.properties
    - datasource.url=jdbc:postgresql://127.0.0.1:5432/dreamjob
    - datasource.username=postgres
    - datasource.password=password
10. Создать класс конфигурацию ru.job4j.dreamjob.configuration.DatasourceConfiguration
11. Реализовать работу с БД ru.job4j.dreamjob.repository.Sql2oCityRepository
12. Реализовать ru.job4j.dreamjob.repository.Sql2oFileRepository
13. Изменить Vacancy
14. Реализовать ru.job4j.dreamjob.repository.Sql2oVacancyRepository
15. Изменить названия параметров в конструкторах, чтобы использовался созданный репозиторий

## Контекст:
- DatasourceConfiguration
    - Все настройки должны быть в классах-настройках. Соблюдаем SRP;
    - Пул создается в методе connectionPool. В качестве его реализации выступает класс BasicDataSource;
    - Метод databaseClient создает экземпляр Sql2o.
    - createConverters() создает конвертер, который делает преобразование из Timestamp в LocalDateTime и наоборот. Этот конвертер будет использоваться Sql2o.
- Sql2oFileRepository
    - Метод findById() Добавляем параметр с помощью addParameter. Удобный метод executeAndFetchFirst возвращает нам сразу первый элемент из выборки. Не нужно писать if, как с JDBC;
    - Метод deleteById() выполняется с помощью executeUpdate();
    - Метод save(). Во-первых, флаг true, который указывает на необходимость возврата ключа из запроса. getKey() соответственно нам его возвращает. 
- Vacancy
    - В Vacancy нужно написать маппинг в виде мапы COLUMN_MAPPING, ключи которой это столбцы из БД, а значения названия полей.
